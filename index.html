overfitting && underfitting

overfitting

the model is to complex，Recorded data characteristics，Noise and irrelevant details.

The model performs well in the training data，but perform bad in new data.

cannot generalization to new data.

Eg: Predictive house price model，focus on tiny features of each house，such as door handle shape

These characteristics are not universally applicable to predict the price of a new house.

underfitting

The model is too simple, it cannot capture the main characters of the data set.

Eg: Predictive house price model,

The model only consider the house area as the affecting factorignore some other key factors, like location, age.

bias/variance trade off

bias

The difference between predict value and real value

high bias means: the model performs bad in the training data set and test data set.

variance

The sensitivity of the model to the given data，The performance of the model on different data sets.

high variance means: the model is complicated or overfitting

the model shows high performance in train data set but unstable/poor performance on new data or test sets.

trade off

including three part:
bias

​ variance

​ irreducible（不可改变的意思）

The total prediction is to make the sum of these three less.

Irreducible Error

​ randomness or natural variable(随机性或自然变量)

the weather is complicated, it may affected by numerous interacting factors

some of them are unknown or hard-to-quantity

​ measurement errors or data collection inaccuracies(测量错误)

Even the most accurate thermometer has measurement errors.

​ missing data（数据缺失）

Some data cannot put captured

Preventive measures of overfitting

  1. increase training data set

    increase data

  2. data augmentation(数据增强)

    eg: in CV, rotating, scaling, cropping(裁剪),

    or adjust the color of data.

    eg: in NLP, synonym replacement(同义词替换), Sentence rearrangement,

    back-translation(Translate the text into another language,

    and then translate it back to the original language.)

  3. decrease the model complexity

    3.1 choose simple model

    3.2 decrease the number of parameter of the model

  4. Use regularization

    use regularization to make the weights of the model as small as possible

    4.1 L1 regularization

    4.2 L2 regularization

  5. Cross-validation

    Divide the original data into multiple subsets, and then use these subsets to train and verify the model in different ways.

    eg: K-Fold Cross-Validation

TVM

Tensor Virtual Machine

深度学习的编辑器

将深度学习模型部署到硬件平台的 开源编辑器。

通俗解释就是将 模型 转化为 机器码 可以在任何设备上运行。

TVM支持不同的前端(深度学习框架)： ONNX, Pytorch

TVM支持不同的后端(执行代码的硬件平台): CUDA, LLVM(complier 可以用来加速代码的运行速度)

LLVM 支持多个GPU类型上运行代码。

CUDA 只支持 NVIDIA的GPU上进行运行。

举个例子理解

使用PyTorch框架来训练一个深度神经网络模型。在PyTorch中，定义模型的结构，选择优化器，训练模型，并对其进行验证。一旦模型训练完成并验证通过，需要将这个模型部署到不同的硬件上（比如GPU、CPU或专用的加速器上）。

前端就是 pyTorch， 用于 创建 和 训练 模型。

需要将训练好的模型，转变为ONNX格式，让TVM处理。

TVM将模型转为 Relay IR，更加通用。

后端是 在硬件平台上运行。

TVM中选择LLVM作为后端意味着你可以将模型编译为能在各种CPU上运行的代码，而不仅仅是特定类型的GPU。

TVM中选择CUDA，适用于NVIDIA的GPU，需要学习CUDA编程知识，对于深度学习和计算密集型任务会有更好的性能。

Official Website

https://tvm.apache.org/

安装python包

转换格式

可以使用TVM模型转换工具

3. 使用优化器优化模型

TVM自动优化工具： AutoTVM 和 AutoScheduler

自动找到最优的内核配置和计算图优化策略，以提高在特定硬件上的性能

优化内核配置： 特定的任务 使用选择 相应的 处理器内核，从而提升性能。

CPU GPU FPGA

硬件平台： 是支持软件运行和执行的物理组件和设备的集合。

CPU Central processing unit： 负责解释和执行大部分计算机指令。

GPU Graphics processing unit： 最初用于处理图形和图像数据，现在也用于加速并行计算任务。

FPGA Field programmable gate array: 高度定制，适用于特定的硬件。

并行计算任务： 该任务 可以被分解为 多个子任务，斌且在多个处理器上并行执行。

相关类似Item

TensorRT： NVIDIA，专门针对其GPU优化，提升深度学习在GPU运行效率。

ONNX Runtime： 方便模型 在不同框架上的迁移和使用。 通过 ONNX格式表示通用模型，这样不同框架训练的模型都可以在ONNX Runtime上运行。

digital signature and verification 数字签名

Verify the integrity and source of digital information

use AKA asymmetric key algorithm( RSA or ECC) to generate

purpose

  1. make sure the data has not been tampered修改 by unauthorized individuals

  2. Varify the identity of sender

private key && public key 私钥 和 公钥

Asymmetric encryption的basic conception

在 Asymmetric encryption system中， use a key Paris:

one public key and one private key

private key

only the onwer know the private key

Encrypted communication: private key use to decrypt the message

digital signature: private key use to encrypt the hash value of message

public key

The public key is public and can be safely shared with anyone.

Generate digital signatures 生成数字签名

make sure message Authenticity and integrity

使用 asymmetric key algorithm 实现

  1. create hash value

    Use hash function SHA-256 to handle message

    genrate a fixed size hash value

  2. Encrypt hash value

    Sender use private key to encrypt the hash value

    the encry value will be the digital signature

  3. add signature

    sender attach the digital signature after the message (一般来说是发送 保密信息)

    or

    sender send to the recevier separately (发送软件 or 系统)

Verify digital signature 验证数字签名

  1. get signature and data
  2. Decrypt signature
  3. Recalculate hash value
  4. compare hash value

将recalcuate hash value 和 decrypt hash value比较

两个hash value一致，说明消息没有被tampered修改，而且是由相应的Private key holder发送的

  1. 初始化OpenSSL，生成RSA key pair
  2. Sign a sample message
  3. Verify this signature.
